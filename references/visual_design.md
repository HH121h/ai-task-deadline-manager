# AI 作业截止日管理器 — 可视化设计规范

## 设计原则

### 简约清楚
- **大量留白**：卡片间距 ≥ 20px，内边距 ≥ 20px
- **克制色彩**：全站使用 5 种语义色
- **一目了然**：每个卡片只传达一个核心信息

### 配色方案
| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | #3B82F6 | 一般任务、链接、按钮 |
| 警示 | #F59E0B | 2天内截止、中等紧急 |
| 危险 | #EF4444 | 逾期、今日截止、高紧急 |
| 成功 | #10B981 | 已完成、准确估算 |
| 中性 | #64748B | 次要文字、标签 |
| 背景 | #F8FAFC | 页面背景 |
| 边框 | #E2E8F0 | 分割线、边框 |

### 语义图标
- 📋 今日任务 · 📅 本周总览 · 📊 偏差分析 · 🏆 复盘统计
- 🎯 优先级 · ⏰ 时间/紧急 · 📈 趋势 · 💡 建议 · 🔴 逾期 · ✅ 完成

## 多 Tab 结构

### Tab 1: 今日任务
| 区域 | 类型 | 说明 |
|------|------|------|
| 今日优先顺序 | 卡片列表 + 进度条 | pending 按 priority_score 降序，前3名特殊标记 |
| 紧急度仪表 | 半环形 Gauge | 所有 pending 的紧急度加权分数 |
| 今日难度分布 | 环形图 | 各难度等级任务数 |
| 时间估算甘特 | 双行进度条 | 每任务剩余时间条 + 耗时条 |

### Tab 2: 本周总览
| 区域 | 类型 | 说明 |
|------|------|------|
| 4个统计卡片 | 数字 | 本周完成数 / 待完成预计耗时 / 已投入时间 / 逾期数 |
| 任务完成率 | 环形图 | completed / pending / overdue 比例 |
| 各科目分布 | 饼图 | 按 course 分组 |
| 逾期列表 | 卡片列表 | 按超期天数排序 |

### Tab 3: 偏差分析
| 区域 | 类型 | 说明 |
|------|------|------|
| 各科偏差 | 水平柱状图 | avg_bias，红>1.1 / 橙<0.9 / 绿=准 |
| 偏差趋势 | 折线图 | bias_history 按时间排序，含 y=1.0 基准线 |
| 智能校准建议 | 左侧色条卡片 | 具体调整数字 + 关联待办 |

### Tab 4: 复盘统计
| 区域 | 类型 | 说明 |
|------|------|------|
| 4个统计卡片 | 数字 | 总完成 / 总投入 / 课程数 / 全局偏差 |
| 各科耗时排名 | 水平柱状图 | 按课程聚合 actual_hours |
| 完成趋势 | 折线图 | 按月聚合 completed 数 |

## 图表交互规范
- 所有图表使用 Chart.js，CDN 加载
- Tab 切换后 100ms 调用 chart.resize()
- 无数据时显示 empty-state（🎉 / 📅 / 📊）
- 响应式：>768px 多列，≤768px 单列
